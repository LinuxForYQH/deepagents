🎯 项目简介

Deep Agents 是 LangChain 团队开发的开箱即用智能代理框架。借鉴 Claude Code、Deep Research、Manus 等产品理念，通过四大支柱实现深层任务处理。

📋

规划 (Planning)

write_todos 让代理自主分解任务、跟踪进度

📁

文件系统 (Filesystem)

read_file、write_file、edit_file、ls、glob、grep

🤖

子代理 (Sub-agents)

task 工具支持将复杂任务委派给独立子代理

💡

智能提示 (Smart Prompts)

内置精心设计的系统提示，教会模型有效使用工具

💡 核心定位

一个有主见的、开箱即用的代理框架。无需自己编排提示词、工具和上下文管理——直接获得一个可工作的代理，按需定制。

🏗️ 架构总览

⚙️ create_deep_agent()

核心入口函数，位于 deepagents/graph.py。返回 CompiledStateGraph，可直接 invoke/stream。

def create_deep_agent(
    model: str | BaseChatModel | None = None,
    tools: Sequence[BaseTool | Callable | dict] | None = None,
    *, # 以下均为关键字参数
    system_prompt: str | SystemMessage | None = None,
    middleware: Sequence[AgentMiddleware] = (),
    subagents: Sequence[SubAgent | CompiledSubAgent | AsyncSubAgent] | None = None,

    memory: list[str] | None = None,
    backend: BackendProtocol | BackendFactory | None = None,
    checkpointer: Checkpointer | None = None,
    store: BaseStore | None = None,
    interrupt_on: dict[str, bool | InterruptOnConfig] | None = None,
    debug: bool = False,
    name: str | None = None,
    cache: BaseCache | None = None,
) -> CompiledStateGraph

💾 Backend 存储层

所有 Backend 实现统一的 BackendProtocol 接口：

方法功能异步版
ls(path)列出目录als()
read(path, offset, limit)读取文件（分页）aread()
write(path, content)创建新文件awrite()
edit(path, old, new)字符串替换aedit()
grep(pattern, path, glob)文本搜索agrep()
glob(pattern, path)模式匹配aglob()
upload_files()批量上传aupload_files()
download_files()批量下载adownload_files()

各类 Backend

StateBackend 默认临时

文件存储在 LangGraph state 中，随对话持续但不跨线程

FilesystemBackend 本地

直接读写文件系统。支持 root_dir、virtual_mode。Grep 优先 ripgrep

StoreBackend 持久化

基于 LangGraph BaseStore，跨线程持久化，用 namespace 隔离

CompositeBackend 路由

按路径前缀路由到不同 backend

LocalShellBackend Shell

继承 FilesystemBackend + 本地 shell 执行（无沙箱）

LangSmithSandbox 云端

基于 LangSmith SDK 的云端沙箱

from deepagents.backends import FilesystemBackend

backend = FilesystemBackend(root_dir="./project", virtual_mode=True)
agent = create_deep_agent(backend=backend)
⚠️ 安全警告

FilesystemBackend/LocalShellBackend 赋予代理直接文件系统访问权限。建议配合 HITL 中间件或在沙箱中使用。

🔌 中间件体系

中间件能拦截每次 LLM 调用，动态修改工具列表、注入系统提示、管理跨轮次状态——这是普通工具函数做不到的。

为什么用中间件？

① 动态过滤工具列表 ② 注入系统提示上下文 ③ 转换消息（上下文压缩） ④ 维护跨轮次状态。工具只能被 LLM 调用，不能拦截 LLM 调用。

中间件列表

中间件职责
FilesystemMiddleware注册 ls/read_file/write_file/edit_file/grep/glob/execute 工具，根据 backend 动态过滤
MemoryMiddleware加载 AGENTS.md 内容，注入到 <agent_memory> 标签，含学习反馈指导
SkillsMiddleware加载 SKILL.md（YAML frontmatter），渐进式披露：列表仅显示名称+描述
SubAgentMiddleware注册 task 工具，管理同步子代理，状态隔离
AsyncSubAgentMiddleware管理远程异步子代理，通过 LangGraph SDK 启动后台任务
SummarizationMiddleware自动对话压缩 + compact_conversation 手动触发工具
PatchToolCallsMiddleware为缺少 ToolMessage 的 tool_calls 补充"已取消"消息
TodoListMiddleware提供 write_todos 规划工具（来自 langchain.agents）

🔬 示例一：Deep Research — 深度网络研究

一个多步骤深度研究代理，使用 Tavily 作为 URL 发现引擎（非摘要引擎），配合 httpx 全文抓取 + markdownify 转换，并通过 think_tool 实现战略反思。

架构设计

编排器 (claude-sonnet-4-5) │ RESEARCH_WORKFLOW_INSTRUCTIONS + SUBAGENT_DELEGATION_INSTRUCTIONS │ ├── tavily_search URL 发现 → httpx 全文抓取 → markdownify 转 Markdown ├── think_tool 战略反思：分析发现 → 识别差距 → 规划下一步 │ └── 子代理: research-agent (最多3个并行, 最多3轮) │ RESEARCHER_INSTRUCTIONS ├── tavily_search └── think_tool

核心代码

from deepagents import create_deep_agent
from langchain.chat_models import init_chat_model

# 编排器指令 = 研究工作流 + 子代理委托策略
INSTRUCTIONS = RESEARCH_WORKFLOW_INSTRUCTIONS + SUBAGENT_DELEGATION_INSTRUCTIONS

# 定义研究子代理
research_sub_agent = {
    "name": "research-agent",
    "description": "Delegate research to the sub-agent researcher.",
    "system_prompt": RESEARCHER_INSTRUCTIONS.format(date=current_date),
    "tools": [tavily_search, think_tool],
}

model = init_chat_model(model="anthropic:claude-sonnet-4-5-20250929", temperature=0.0)

agent = create_deep_agent(
    model=model,
    tools=[tavily_search, think_tool],
    system_prompt=INSTRUCTIONS,
    subagents=[research_sub_agent],
)

自定义工具详解

tavily_search — URL 发现 + 全文抓取

@tool(parse_docstring=True)
def tavily_search(
    query: str,
    max_results: Annotated[int, InjectedToolArg] = 1,      # 对 LLM 隐藏
    topic: Annotated[Literal[...], InjectedToolArg] = "general",
) -> str:
    # 1. Tavily 发现 URL（不使用其摘要）
    search_results = tavily_client.search(query, max_results=max_results)
    # 2. httpx 抓取完整网页
    content = fetch_webpage_content(url)  # → markdownify 转 Markdown
    return formatted_results

think_tool — 战略反思

@tool(parse_docstring=True)
def think_tool(reflection: str) -> str:
    """让代理暂停评估进度、分析差距、规划下一步。"""
    return f"Reflection recorded: {reflection}"

三层 Prompt 架构

提示词作用对象内容
RESEARCH_WORKFLOW编排器5 步工作流：保存请求 → 规划 TODO → 委托子代理 → 综合 → 写报告
SUBAGENT_DELEGATION编排器委托策略：简单查询 1 个子代理，比较类 1 个/元素，最多 3 并行
RESEARCHER_INSTRUCTIONS研究子代理搜索预算（简单 2-3 次，复杂最多 5 次），每次搜索后使用 think_tool 反思

# 加载 YAML 并映射工具名到实际对象
def load_subagents(config_path):
    available_tools = {"web_search": web_search}
    with open(config_path) as f:
        config = yaml.safe_load(f)
    subagents = []
    for name, spec in config.items():
        subagent = {"name": name, "description": spec["description"], ...}
        if "tools" in spec:
            subagent["tools"] = [available_tools[t] for t in spec["tools"]]
        subagents.append(subagent)
    return subagents

自定义工具 — Gemini 图片生成

@tool
def generate_cover(prompt: str, slug: str) -> str:
    """生成博客封面图。保存到 blogs/<slug>/hero.png"""
    from google import genai
    client = genai.Client()
    response = client.models.generate_content(
        model="gemini-2.5-flash-image", contents=[prompt])
    for part in response.parts:
        if part.inline_data is not None:
            image = part.as_image()
            image.save(f"blogs/{slug}/hero.png")
    return "Image saved"

核心代码

from deepagents import create_deep_agent
from deepagents.backends import FilesystemBackend
from langchain_anthropic import ChatAnthropic
from langchain_community.agent_toolkits import SQLDatabaseToolkit
from langchain_community.utilities import SQLDatabase

def create_sql_deep_agent():
    db = SQLDatabase.from_uri(f"sqlite:///{db_path}", sample_rows_in_table_info=3)
    model = ChatAnthropic(model="claude-sonnet-4-5-20250929", temperature=0)

    # LangChain SQL 工具集: list_tables, schema, query, query_checker
    toolkit = SQLDatabaseToolkit(db=db, llm=model)
    sql_tools = toolkit.get_tools()

    agent = create_deep_agent(
        model=model,
        memory=["./AGENTS.md"],          # 身份 + 安全规则
        skills=["./skills/"],             # 查询编写 + Schema 探索
        tools=sql_tools,                 # SQL 工具集
        subagents=[],                    # 不需要子代理
        backend=FilesystemBackend(root_dir=base_dir),
    )
    return agent

🔄 示例五：Ralph Mode — 自主循环模式

受 Geoff Huntley 启发的自主循环模式：每次迭代以全新上下文启动代理，文件系统和 git 作为跨迭代的持久化记忆。原始实现仅一行：while :; do cat PROMPT.md | agent ; done

核心原理

Ralph 循环 ┌─────────────────────────────────────────────────┐ │ Iteration 1 (全新上下文) │ │ "Your previous work is in the filesystem. │ │ Check what exists and keep building. │ │ TASK: Build a Python course. Make progress." │ │ ↓ │ │ Agent 执行 → 创建/修改文件 → git commit │ └─────────────────────────────────────────────────┘ ↓ 文件系统持久化 ┌─────────────────────────────────────────────────┐ │ Iteration 2 (全新上下文，但文件系统保留) │ │ Agent 检查已有文件 → 继续推进 → 保存进度 │ └─────────────────────────────────────────────────┘ ↓ ... ↓ Ctrl+C 停止 → 列出所有生成文件

核心代码

from deepagents_cli.non_interactive import run_non_interactive

async def ralph(task, max_iterations=0, model_name=None, ...):
    iteration = 1
    while max_iterations == 0 or iteration <= max_iterations:
        prompt = (
            f"## Ralph Iteration {iteration}\n\n"
            f"Your previous work is in the filesystem. "
            f"Check what exists and keep building.\n\n"
            f"TASK:\n{task}\n\nMake progress. You'll be called again."
        )
        exit_code = await run_non_interactive(
            message=prompt,
            assistant_id="ralph",
            model_name=model_name,
            sandbox_type=sandbox_type,
            quiet=True, stream=stream,
        )
        if exit_code == 130: break  # Ctrl+C
        iteration += 1

🔷 Agent 设计模式

1. ReAct Agent — 从零构建

最基础的 Agent 模式：LLM 推理→调用工具→观察结果→循环直到得出答案。

START → agent (LLM 决策) → [条件: 有工具调用?] │ yes → tools (执行工具) → agent (循环) │ no → END

变体：ReAct + 结构化输出 — 使用 Pydantic 模型约束 Agent 的最终输出格式。

2. Plan-and-Execute — 先计划后执行

先一次性生成完整步骤计划，然后逐步执行每一步，并根据执行结果重新规划。

START → Planner (生成步骤列表) │ ↓ Executor (执行当前步骤) │ ↓ Re-planner (根据结果更新计划) → [条件: 还有步骤?] │ yes → Executor (循环) │ no → END

3. LLM Compiler — 并行工具执行

LLM 作为任务规划器，生成 DAG 形式的工具调用计划，独立调用并行执行。

START → Planner (生成 DAG 调用计划) │ ↓ Task Fetching (并行执行独立工具调用) │ ↓ Joiner (合并结果) → [条件: 需要重新规划?] │ yes → Planner (循环) │ no → END

4. Multi-Agent 协作

模式描述关键技术
扁平协作多个专业 Agent（Researcher, Coder 等）通过共享消息状态协作路由器条件边，共享 messages 状态
层级化团队顶层 Supervisor → 子团队 Supervisor → 专业 Agent，支持嵌套嵌套 StateGraph（子图），层级路由

5. ReWOO — 无观察推理

与 ReAct 不同，ReWOO 先一次性制定完整计划（含所有工具调用），然后批量执行，最后用全部结果推理。减少 LLM 调用次数。

START → Planner (一次性规划所有步骤 + 工具调用) │ ↓ Worker (批量执行所有工具调用) │ ↓ Solver (综合所有结果进行推理) → END

🔷 RAG 变体详解 (4 种核心架构)

LangGraph 最丰富的示例类别，包含 4 种 RAG 架构（含本地 LLM 变体共 9 个 notebook）。

CRAG — 纠正式 RAG

论文：arxiv.org/abs/2401.15884

检索后对文档进行相关性评分，若文档不相关则改写查询 + Web 搜索补充。

START → retrieve → grade_documents → [条件: 文档相关?] │ yes → generate → END │ no → transform_query → web_search → generate → END
class GraphState(TypedDict):
    question: str
    generation: str
    web_search: str      # "yes" / "no" 标志
    documents: List[str]

Self-RAG — 自反思 RAG

论文：arxiv.org/abs/2310.11511

比 CRAG 多两层检查：幻觉检测（生成是否基于文档）+ 答案质量检查（是否真正回答了问题）。

START → retrieve → grade_documents → [条件: 文档相关?] │ yes → generate → [条件: 幻觉检查 + 答案质量] │ │ 通过 → END │ │ 幻觉 → generate (重试) │ │ 不相关 → transform_query → retrieve (循环) │ no → transform_query → retrieve (循环)

Adaptive RAG — 自适应 RAG

论文：arxiv.org/abs/2403.14403

统一查询分析 + 自纠正 RAG。先通过路由器分析问题该走 Web 搜索还是向量库检索，检索后走 Self-RAG 全套自纠正。

START → [路由: route_question] │ web_search → web_search → generate → END │ vectorstore → retrieve → grade_documents → [条件] │ yes → generate → [幻觉+质量检查] │ no → transform_query → retrieve

Agentic RAG — Agent 驱动 RAG

将 retriever 包装为工具，让 Agent 自主决定是否需要检索。使用 add_messages 注解的消息追加模式。

START → agent (LLM 决策) → [条件: 需要检索?] │ yes → retrieve (ToolNode) → grade_documents → [条件] │ │ 相关 → generate → END │ │ 不相关 → rewrite → agent (循环) │ no → END
class AgentState(TypedDict):
    messages: Annotated[Sequence[BaseMessage], add_messages]  # 消息追加

🔷 研究论文实现

Reflection 基础反思

Generator 生成 → Reflector 批评 → 基于反思修改 → 循环直到满意。最基础的自我改进模式。

Reflexion 论文

Agent 执行→环境反馈→语言化反思→存入反思记忆→后续迭代使用。比基础反思更进一步。

LATS 树搜索

LLM + 蒙特卡洛树搜索（MCTS），在树结构中探索多条推理路径，反思+回溯找最优答案。

Self-Discover 论文

LLM 自动发现推理结构：① SELECT 推理模块 ② ADAPT 调整 ③ IMPLEMENT 构建 ④ SOLVE 解题。线性 Pipeline。

🧩 设计理念

四大支柱

规划工具 + 子代理 + 文件系统 + 详细提示 — 借鉴 Deep Research / Manus / Claude Code

中间件优先

中间件拦截每次 LLM 调用，能动态修改工具列表、注入提示、管理状态 — 普通工具做不到

Backend 可插拔

统一接口支持临时状态、本地文件、持久化存储、云沙箱等多种策略

子代理隔离

独立上下文窗口和状态，只返回最终结果，防止主线程上下文膨胀

渐进式披露

技能系统仅在提示中显示名称和描述，agent 按需读取完整指令，节省 token

LangGraph 原生

返回 CompiledStateGraph，兼容 streaming、Studio、checkpointers 等所有 LangGraph 功能
